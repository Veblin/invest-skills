# 评估细则 + 校准场景

> v0.2.1 参考文档。供 Claude 在评估时按需加载。

## 一、买入四维评估细则

### 1. 逻辑完整性（Logic）

| 子维度 | ✅ 通过条件 | ⚠️ 需改进 | ❌ 不通过 |
|--------|-----------|-----------|---------|
| 驱动逻辑 | 具体可验证的因果关系 | 方向对但笼统 | 模糊或循环论证 |
| 核心假设 | 可量化的前提条件 | 定性但合理 | 无假设或"会涨" |
| 错误条件 | 有明确触发阈值 + 覆盖主要风险 | 有但阈值模糊 | 无错误条件 |
| 假设-逻辑一致性 | 驱动逻辑和假设互相支撑 | 部分自洽 | 自相矛盾 |

### 2. 数据盲点（Blind Spots）

自动检查 8 项（基于 query_data + market_microstructure 输出）：

1. PE 分位：是否被提及？无 Tushare → 标注 "无历史分位"
2. 对冲工具：ETF → 查 hedge_coverage；个股 → 多数无对冲
3. 拥挤度：如果涉及科技/小盘 → 标注 "未查询"
4. 流动性：日成交额趋势
5. 宏观环境：PMI/LPR/VIX 是否支持假设
6. 杠杆水平：两融/流通市值 + 标签
7. 市场广度：涨跌比
8. 极端情绪：涨跌停比 + 跌停家数

每项缺失的数据 → 追加 blind_spot note。

### 3. 仓位匹配（Position Sizing）

| 检查项 | 方法 |
|--------|------|
| 波动率 × 仓位 | 年化波动率 × 仓位% ≈ 组合风险贡献。与用户声明的最大可接受损失对比 |
| 对冲调整 | 有对冲工具（coverage=high）→ 仓位弹性更大；无对冲（coverage=none）→ 裸多头风险 |
| 名义 vs 可退出 | 跌停 >50 家 → 名义仓位 ≠ 可退出仓位 |

输出格式：陈述事实，追问矛盾，不给 "建议 ≤X%"。

### 4. 风险收益比（Risk/Reward）

- 下方空间估算：PE 回归中位数 + 盈利下行 → 价格区间
- 上方空间估算：基于用户假设的盈利路径 → 价格区间
- 非对称性判断：下方 > 上方 → 赚小钱冒大险

## 二、卖出三维评估细则

### 1. 一致性（Consistency）

- 对比入场 PE/价格 vs 当前 PE/价格
- 当初的错误条件触发了吗？
- 条件未触发却要卖出 → 追问理由

### 2. 情绪检测（Emotion Check）

- 卖出是否发生在连续大跌后（>15%/5日）→ 恐慌性割肉风险
- 卖出理由中是否有情绪词（"害怕""受不了""感觉"）
- 涨跌比 <0.4 + 跌停 >50 → 大概率恐慌

### 3. 机会成本（Opportunity Cost）

- 钱出来后去哪？持币还是换仓？
- 两融/市值趋势：去杠杆 → 持币观望可能更合理；冰点 → 可能错过反弹

## 三、校准场景

### 场景 1：中证 2000 ETF 无对冲高仓位（买入）

**用户方案**：买入 563300 中证 2000 ETF，仓位 8%，驱动逻辑 "小盘因子均值回归"，最大损失 10%。

**数据假设**：PE 38.5x（csindex），折价 -0.5%，AUM 15 亿，无对冲工具，年化波动率 31.5%，涨跌比 1.2。

**期望评估结果**：
- 逻辑: ⚠️（"均值回归" 从哪里回归？无具体锚点）
- 盲点: ❌（无对冲工具未被识别 + PE 分位可能偏高）
- 仓位: ❌（31.5% 波动 × 8% 仓位 × 无对冲 ≈ 组合风险过高）
- 风险收益: ⚠️（需计算具体下方空间）

**护栏追加**：若两融=去杠杆 → 追加 "流动性收紧" 盲点。

### 场景 2：ETF 溢价买入（ETF 买入）

**用户方案**：买入某 ETF，驱动逻辑 "行业景气反转"，未注意溢价。

**数据假设**：溢价 +3.2%，AUM 80 亿，有对冲。

**期望评估结果**：
- 盲点: ⚠️（溢价 3.2% → 买入成本偏高，需要时间消化溢价回归）
- etf_data.py flags 自动触发了 "⚠️ 溢价 3.2%" 标记
- 评估应追问 "你知道现在溢价 3.2% 吗？"

### 场景 3：7 月 V 型反弹中追涨（2026-07-21 基准案例）

**数据快照（7/21）**：
- 涨跌比 1.35（正常偏暖）
- 涨跌停比 9.8:1（极端亢奋）
- 两融/市值 ~2.7% 且连降（去杠杆中）

**护栏预期输出**：
- 规则 1 触发（去杠杆进行中）→ 追加盲点
- 规则 2 触发（涨跌停比 >5:1）→ 追加情绪回归风险
- 规则 3 不触发（广度正常）
- 两个盲点同时出现 → "去杠杆 + 亢奋 = 背离"

## 四、边界条件

### 数据完全不可得（全部 missing）

用户说 "我想买 XXX"，但 query_data 全部返回 missing。

→ 评估输出逻辑完整性基于用户文字（不依赖数据）；其他三维标注 "数据不可得，无法评估"；不阻止保存。

### 只有腾讯行情（无 Tushare）

PE 分位 = missing。标注 "当前 PE 36.5x（腾讯行情），无历史分位。建议后续通过 Tushare 补充分位数据。"

### 标的代码无效

直接告知用户 "未找到代码 XXX 的数据，请确认代码是否正确？"。不继续评估。
