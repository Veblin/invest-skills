## 4. 执行顺序建议

```
Phase 0 ─→ Phase 1 ─→ Phase 2 ─→ Phase 3 ─→ Phase 4 ─→ Phase 5
  (1-2h)    (10-14h)    (7-9h)     (5-7h)     (4-6h)     (2-3h)

Phase 4 → Phase 5 必须串行：Phase 5 的测试矩阵覆盖 Phase 4 的对比/宏观/教学模式。
```

**关键里程碑**：
- Phase 1.10 完成 → 首次产生结构化数据
- Phase 2.4 完成 → 首次生成完整报告
- Phase 3.3 完成 → `/invest-A` 端到端可用

**执行原则**：遵循 [执行步骤](./执行计划-执行步骤.md) 的逐步骤顺序，每步 ≤2h，审阅签字后再进入下一步。
