## 3. 七维度 ↔ 策略 ↔ 数据模块映射

与《股票 ETF 调研数据采集框架》七大维度对齐。报告按此顺序展开；采集优先级见框架文档末尾排序（财务优先，情绪靠后）。

| # | 框架维度 | 策略 YAML | 主要数据模块 | MVP |
|---|---------|-----------|-------------|-----|
| 一 | 官方基本信息 | `basic_info.yaml` | `a_share_data` / `hk_share_data` | ✅ |
| 二 | 财务报告 ★★★★★ | `fundamental.yaml` | `get_financials` + Tushare | ✅ |
| 三 | 行业与产业链 ★★★★☆ | `industry.yaml` | `news_search` + `industry_chain`（图谱 Phase 6） | ⚠️ 图谱延后 |
| 四 | 市场交易数据 | `valuation.yaml` + `technical.yaml` | K 线、估值、资金流；ETF 见 Phase 6 | ⚠️ ETF 延后 |
| 五 | 机构与分析师 ★★★★☆ | `institutional.yaml` | `fund_flow`、机构调研（部分 Phase 6） | ⚠️ 一致预期延后 |
| 六 | 情绪与舆情 | `sentiment.yaml` | `sentiment_data` + `news_search` | ✅ |
| 七 | 宏观与政策 | `macro.yaml` | `global_macro` + 政策新闻 | ✅ |
| — | **ETF 专项**（跨维度） | `etf_analysis.yaml` | `etf_data.py` | ❌ Phase 6 |

## 6. 硬约束合规（Phase 5 必过）

对照分析报告 §〇 五条 + 框架文档三条核心原则（后者为前者子集）：

- [ ] **约束 1**：无买入/卖出/持有/目标价/仓位建议
- [ ] **约束 2**：财务数字来自脚本/MCP，非 LLM 记忆
- [ ] **约束 3**：论述标注来源或「待验证」
- [ ] **约束 4**：文档无社交/建群入口
- [ ] **约束 5**：功能从个人学习需求出发，具备分发文档
- [ ] **技术指标**：MA/MACD 仅描述状态，无交易信号（`technical.yaml` + AGENTS.md）

## 10. 执行顺序建议

```
Phase 0 → Phase 1（含 data_pipeline）→ Phase 2（策略与报告）
    → Phase 3（串联工作流）→ Phase 4（增强）→ Phase 5（验收）
    → 按需启动 Phase 6 子项
```

每 Phase 结束应满足该 Phase「验证标准」再进入下一阶段；Phase 1 的 `data_pipeline.py` 是 Phase 3 的前置依赖。
