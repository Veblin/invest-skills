## 二、架构升级：三层产品

不是补章节，而是把整份报告结构改造成**三层嵌套**：

```
┌─────────────────────────────────────────────────────┐
│  🏗️ 第三层：学习层                                   │
│  每节末尾 → "为什么重要" + "常见误判" + "下一步验证"  │
│  受 LAW 10 约束：必须引用本次数据，不得输出通用知识    │
├─────────────────────────────────────────────────────┤
│  🧠 第二层：分析层                                   │
│  固定框架填内容，不是自由发挥                         │
│  受 LAW 14 约束：必须逐项回答，不得合并、跳过或替换    │
│  受 LAW 12 约束：每个结论附带统一强度标注             │
│  行业 = 景气度-竞争格局-议价权-替代威胁（10 题）      │
│  商业 = 护城河-商业模式-成长驱动力（8 题）            │
│  财务 = 增长-盈利-现金流-资本结构（13 题）            │
│  估值 = 绝对位置-相对位置-隐性预期-预期差（10 题）    │
├─────────────────────────────────────────────────────┤
│  📊 第一层：事实层（当前引擎擅长的）                   │
│  基本信息 / 财务 / 估值 / 行业 / 事件 / 技术          │
│  采干净、标清楚、可追溯                              │
│  受 LAW 7 约束：每个数字标注可审查的追溯路径          │
└─────────────────────────────────────────────────────┘
```

每份报告同时包含这三层。用户可以从事实层开始，可以跳进分析框架，也可以只看学习提示。

## 四、三个签名级新增设计

### 1. 研究问题卡（Research Question Tree）

> 受 LAW 11 约束：来自结构化触发，不得凭空总结

**三类触发源（自动检测，优先排序）：**

```
触发源 A — 变化驱动
  → 股价涨跌幅超 ±10% / 核心指标环比/同比变化超 ±5pp / 近 30 日重大公告
  → 问题格式：变化 → 驱动力 → 哪个解释不确定

触发源 B — 估值位置驱动
  → PE/PB 处于历史 80% 分位以上或 20% 分位以下
  → 问题格式：估值在哪 → 隐含预期 → 现实是否支撑

触发源 C — 行业结构驱动
  → 行业指数跑赢/跑输大盘超 5% / 重大行业政策/竞争事件
  → 问题格式：行业变化 → 影响路径 → 是否改变投资逻辑
```

**问题树固定格式：**

```
核心问题（1 句话，问号结尾，含不确定性）
└── 子问题 ①（验证关键数据点）
└── 子问题 ②（挑战核心问题的反证）
└── 子问题 ③（成立时的估值传导路径）

为什么这是好问题：[与估值定价的关系]
```

**禁止：** 宽泛问题 / 无不确定性的问题 / 含买卖判断的问题

### 2. 证据-结论映射（Evidence-Conclusion Mapping）

> 受 LAW 12 约束：统一强度定义，不得自创标准

**三级强度标准：**

| 强度 | 符号 | 定义 |
|------|------|------|
| 强 | ✅ | 直接数据支持，来源可追溯，存在清晰因果链，排除主要竞争性解释 |
| 中 | ⚠️ | 数据方向支持，但存在其他合理解释；或来源单一；或时效性有限 |
| 弱 | ❓ | 仅有时间相关性；或来源不可验证；或数据本身需进一步确认 |

**固定格式：**

```
**结论：[一句话]**

支持证据：
  ✅ [数据 + 来源 + 时间]
  ⚠️ [数据 + 为何不够强的说明]
  ❓ [数据 + 不确定点]

综合证据强度：[强 / 中 / 弱]
  [1-2 句解释]
```

**禁止：** 无证据映射 / 模糊词不降强度 / 逻辑推断打 ✅

### 3. 学习提示卡（Learning Tip Card）

> 受 LAW 10 约束：必须具体，不得输出通用知识

```
📖 学习提示

为什么重要：
  [该维度在估值定价/风险判断中的具体传导路径]

初学者最常误判：
  [引用本次至少一个数据点，构造真实误判场景]
  ✅ 合规：引用具体数字+分析，如"ROE 从 28%→24%易被误判为下滑，但杜邦拆解显示利润率稳定"
  ❌ 违规："ROE 下降时要注意可持续性"——通用知识，无数据

下一步验证：
  ① [具体动作：查什么、和什么对比、期望看到什么]
  ② [优先用已有数据可做的动作]
  ③ [需要外部查阅的动作，可选]
```

## 五、分析正文框架落地

> 详细问题集见 `docs/invest-A-SKILL-LAW10-15.md`「分析框架问题集」章节。
> 受 LAW 14 约束：必须逐项回答，数据不足时写明「数据不足，无法回答：[缺少什么]」。

### 4a. 行业位置分析（10 题）

框架：**景气度 — 竞争格局 — 议价权 — 替代威胁**

```
A1. 景气度（2 题）
  ① 行业整体需求处于扩张还是收缩阶段？
  ② 与 12 个月前相比是改善、恶化还是持平？

A2. 竞争格局（3 题）
  ③ CR3/CR5 集中度如何，近年是提升还是下降？
  ④ 公司按营收/市值的行业排名（龙头/挑战者/追赶者/尾部）？
  ⑤ 近年是否有新进入者或大规模产能扩张？

A3. 议价权（3 题）
  ⑥ 对上游供应商的议价能力？
  ⑦ 对下游客户的集中度和替代风险？
  ⑧ 毛利率与行业中位数对比，差异原因？

A4. 替代威胁（2 题）
  ⑨ 已知的技术路线或商业模式替代威胁？
  ⑩ 替代威胁的时间维度（1 年/3 年/更长期）？
```

### 4b. 商业质量分析（8 题）

框架：**护城河 — 商业模式 — 成长驱动力**

```
B1. 护城河（2 题）
  ① 竞争优势来源（规模/网络效应/切换成本/专利/品牌/成本）？
  ② 护城河在过去 3 年被强化、维持还是侵蚀？

B2. 商业模式（3 题）
  ③ 收入结构：各业务/产品线占比是否变化？
  ④ 成本结构：固定/变动成本为主，规模效应是否显现？
  ⑤ 现金流模式：先收款后交付还是先垫资后回款？

B3. 成长驱动力（3 题）
  ⑥ 过去 3 年主要增长驱动力是什么？
  ⑦ 驱动力是否仍在持续？若已减速，下一驱动力是什么？
  ⑧ 数据不足时明确说明
```

### 4c. 财务质量分析（13 题）

框架：**增长 — 盈利 — 现金流 — 资本结构**

```
C1. 增长（3 题）
  ① 近 3 年营收 CAGR？近一年同比加速还是减速？
  ② 营收增长是量价齐升、以量补价还是以价补量？
  ③ 净利润增速与营收增速相比：利润杠杆方向？

C2. 盈利（4 题）
  ④ 毛利率近 3 年趋势？行业性还是公司特异性？
  ⑤ 费用率是否在扩大？
  ⑥ 扣非净利润/净利润：是否存在非经常性损益扭曲？
  ⑦ 杜邦拆解 ROE：利润率×周转率×杠杆 各自的贡献和趋势

C3. 现金流（3 题）
  ⑧ 经营现金流是否持续正值且覆盖净利润？
  ⑨ 应收/存货增长是否显著快于营收增长？
  ⑩ 自由现金流趋势？是否支撑分红/回购？

C4. 资本结构（3 题）
  ⑪ 资产负债率与行业中位数对比？近 3 年趋势？
  ⑫ 利息保障倍数是否健康（>3）？
  ⑬ 大额有息负债的到期结构是否集中？
```

### 4d. 估值与预期分析（10 题）

框架：**绝对位置 — 相对位置 — 隐性预期 — 预期差**

```
D1. 绝对估值位置（3 题）
  ① PE 处于历史哪个分位？（5 年/3 年/1 年窗口）
  ② PB 处于历史哪个分位？
  ③ 定性：昂贵 / 合理 / 偏低（附分位数据）

D2. 相对估值位置（2 题）
  ④ PE 与行业中位数对比：溢价还是折价？
  ⑤ 溢价/折价是历史一贯还是近年显著变化？原因？

D3. 隐性预期（2 题）← LAW 15 强制计算
  ⑥ 计算市场隐含增长率：g ≈ r - 1/PE
     r = 10Y 国债 + 6% ERP（标注来源和假设）
     对比实际 3 年 CAGR
  ⑦ 从隐性预期角度，市场定价偏乐观/中性/偏悲观？

D4. 预期差识别（3 题）
  ⑧ 若有 WebSearch 获得的一致预期，对比实际财报
  ⑨ 若无一致预期：最大的预期差可能存在于哪个指标？
  ⑩ 下一个能更新市场预期的时间节点和事件？
```

## 六、LAW 13：股价变化候选解释

> 受 LAW 13 约束：不得伪装成归因结论

**固定格式：**

```

候选解释（按证据强度排列）：

→ 解释 A：[1 句话描述]

→ 解释 B：[1 句话描述]

```

**禁止：** "股价下跌是因为……"的确定性归因 / 省略"无法解释的部分" / 将时间相关性当因果

## 七、LAW 15：隐性预期差计算

> 受 LAW 15 约束：即便无外部一致预期数据，仍须输出隐性预期差

**标准计算路径（简化版）：**

```
g_implied ≈ r - 1/PE

其中：
  r = 无风险利率 + ERP
  无风险利率 = 10Y 国债收益率（标注来源/日期）
  ERP = 6%（保守基准，可在不同假设下调整）

解读：
  g_implied > 实际 CAGR → 市场定价偏乐观，需验证增长加速依据
  g_implied < 实际 CAGR → 市场定价偏悲观，可能存在低估
  g_implied ≈ 实际 CAGR → 定价基本反映历史增长，关注拐点
```

**禁止：** 以"无一致预期数据"为由跳过 / 不标注 ERP 假设直接输出数字

## 十二、技术限制与风险

1. **学习提示卡的数据来源：** "常见误判"需要引用本次数据——引擎需要将关键数据点暴露给 AI 渲染层。短期方案：AI 分析时从引擎数据卡片取值；中期可维护误判模式库。
2. **证据-结论映射的自动化程度：** 初级阶段是 AI 写作时手动引用引擎数据点；完全自动化留到 v0.1.4。
3. **隐性预期差（LAW 15）：** 需要 10Y 国债收益率（FRED 已有）和行业 PE（Tushare 已有）。简化版 g ≈ r - 1/PE 对高成长/周期公司参考价值有限。
4. **分析框架 41 题的数据覆盖率：** 部分问题（如客户集中度、CR3/CR5）目前无自动数据源，需要在报告中标注"数据不足，无法回答"。
5. **向后兼容：** 所有新增 `_section_*` 函数返回 Markdown 字符串，与当前 `render_report_v2()` 拼接方式一致。HTML 渲染 `render_html()` 需同步更新。
6. **报告长度增长：** v0.1.2 报告约 300-500 行 Markdown。加入 41 题逐项回答 + 学习提示后，预计增至 800-1200 行。需考虑导航设计。

## 十三、总结

```
v0.1.2 → v0.1.3 的核心转变：

"用户为什么看这份报告" → 研究问题卡（LAW 11：结构化触发，不凭空总结）
"看时该注意什么"       → 学习提示卡（LAW 10：引用本次数据，不输出通用知识）
"结论稳不稳"           → 证据-结论映射（LAW 12：统一三级强度 ✅⚠️❓）
"看完该学会什么"       → 41 题逐项回答 + 三层分析框架 + 可复用方法论（LAW 14）
"为什么涨跌"           → 候选解释（LAW 13：不伪装成归因，诚实标注证据强度）
"值不值这个价"         → 隐性预期差（LAW 15：戈登模型反推，不依赖外部 API）

不是更多章节，而是每节长出"教的能力"。
不是更多数据，而是每句话有证据强度。
不是更聪明的 AI，而是 AI 被框架约束后的"稳定可审计"。
```

**最优先做的一件事：把 `SKILL.md` 写入 LAW 10–15，把 `render_report_v2()` 重构为 8 模块 + 每节三层结构。** 这不需要任何新数据源，只需要改造现有的渲染逻辑。改完之后报告骨架就长出了"教学能力"，后面加数据只是往骨架上填肉。
