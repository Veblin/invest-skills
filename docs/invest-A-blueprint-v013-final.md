## Skill 身份声明

> 本 Skill 是一个**多因子交叉验证的结构化投研助手**。
>
> **它做的：**
> 采集多渠道数据并交叉验证 / 归因分析当前涨跌的驱动因子 /
> 多维度市场状态扫描（行业景气、资金态度、情绪、技术结构）/
> 呈现左侧/右侧概率结构及走势转变触发条件 /
> 所有结论标注来源和证据强度，可追溯可审计。
>
> **它不做的：**
> 不输出个股买卖建议 / 不给目标价 / 不给仓位建议 / 不承诺收益。
> 所有输出限定于"信息整理、结构化分析、多因子交叉验证和归因讨论"，
> 不触碰"买什么 / 什么时候买"的决策边界。
>
> 本 Skill 按阶段逐步增强数据覆盖、交叉验证和归因框架，
> 所有版本变更在 `CHANGELOG` 中可审计。

## 一、审计结论：终稿对原蓝图的三处核心修正

### 修正 1：定位从"学习器"到"投研助手"

原蓝图大量使用"学习提示卡""初学者误判""教的能力"等语言，
导致报告语气偏向教材，而非研究备忘录。

修正方向：

| 维度 | 原语言（删除）| 修订语言（使用）|
|---|---|---|
| 报告语气 | 学习提示 / 学习闭环 | 分析提示 / 研究备忘录 |
| 用户称谓 | 初学者 / 来学习的用户 | 分析者 / 研究者 |
| 输出框架 | 教材式逐节解释 | 专业投研的多因子交叉验证 |
| 结论导向 | "下一步验证什么" | "左侧/右侧概率分布，触发条件是什么" |

---

### 修正 2：补齐"解释当下"的核心能力层

原蓝图四个分析模块（行业/商业/财务/估值）是**静态基本面扫描**，
擅长回答"这家公司好不好"，但无法回答：

- 为什么现在涨跌（短中期驱动因子）
- 行业情绪和资金在哪边（市场结构）
- 左侧还是右侧（趋势位置）
- 什么条件会触发转变（催化剂识别）

修正方向：新增模块 E（动态驱动分析）和模块 F（左侧/右侧概率判断）。

---

### 修正 3：41 题强制执行改为分层激活

LAW 14 强制 41 题容易被"数据不足，无法回答"批量填充，形式合规但质量低。

修正方向：每模块设 3-4 道核心必答题（共 12 题），
扩展题由研究问题卡的触发条件动态激活。

```

   强度：✅ / ⚠️ / ❓

   强度：✅ / ⚠️ / ❓

⚠️ 尚无候选解释的部分：[说明哪段变化无法解释 + 缺乏什么数据]

### 多因子驱动矩阵

| 因子类别 | 具体信号 | 方向 | 强度 | 数据来源 |
|---------|---------|------|------|---------|

| 资金（主力） | 近 5 日主力大单净额 | | | Tushare moneyflow（2000积分）|
| 情绪（融资） | 融资余额近 10 日变化 | | | Tushare margin（2000积分）|
| 情绪（换手） | 近 5 日换手率 vs 60 日均值 | | | Tushare daily_basic |
| 技术趋势 | MA5/MA20/MA60 多空排列 | | | 引擎计算 |
| 事件催化 | 近 30 日重大公告/政策 | | | Tushare anns / WebSearch |

---

### 因子交叉验证结论

因子方向一致性：[X 个正向 / Y 个负向 / Z 个中性]

若多数因子方向一致：
  → 主导因子：[哪个因子当前解释力最高，依据是什么]
  → 驱动一致性：强 / 中 / 弱
  → 主导因子的可持续性：短期（1-5 交易日）/ 中期（1-3 个月）/ 待观察

若因子方向相互矛盾：
  → 分歧点：[哪些因子矛盾，可能的解释是什么]
  → 当前主导因子：[短期内对股价解释力更高的因子，为什么]
```

### 5.3 多数据渠道交叉验证设计

贯穿全报告的标注机制：

```
🟢 印证（Convergence）：多个独立数据源指向同一结论
   → 证据强度提升，结论可信度高

🟡 分歧（Divergence）：不同数据源给出矛盾信号
   → 必须说明：哪两个数据源矛盾 / 可能的解释 / 哪个更可靠

🔴 缺口（Gap）：某个关键维度无法获得独立验证
   → 必须标注：缺少什么数据 / 该维度结论的可信度降低
```

**报告中的具体应用位置：**

| 模块 | 交叉验证的数据对 |
|------|----------------|
| 财务质量 | 净利润 vs 经营现金流（利润含金量验证）|
| 财务质量 | 营收增长 vs 应收账款增长（收入确认质量）|
| 估值 | PE 历史分位 vs PB 历史分位（估值一致性）|
| 资金态度 | 北向净流入 vs 主力大单净额（资金方向一致性）|
| 行业情绪 | 申万板块指数 vs 个股相对强弱（公司特异性验证）|
| 技术 vs 基本面 | MA 趋势方向 vs 近期业绩方向（趋势与基本面一致性）|
| 估值 vs 资金 | PE 历史低位 vs 资金持续流出（低估陷阱识别）|
| 情绪（左侧）| ERP 分位 vs 认沽认购比 vs 融券余额增速（情绪一致性）|

## 六、数据接口与权限说明（补充调研结论）

这是原蓝图中最模糊的地方，用调研结果补全。

### 6.1 Tushare 接口权限分层

| 接口 | 所需积分 | 说明 |
|------|---------|------|
| `daily_basic`（换手率、PE/PB）| 2000 | 基础行情衍生指标 |
| `stk_factor`（技术指标）| 2000 | 含 MA、MACD、RSI 等 |
| `fina_indicator`（财务指标）| 2000 | ROE、毛利率、现金流 |
| `moneyflow`（个股资金流向）| 2000 | 大单/中单/小单净额 |
| `hsgt_top10`（沪深股通十大成交）| 2000 | 北向个股交易数据 |
| `margin`（融资融券数据）| 2000 | 融资余额、融券余额 |
| `sw_daily`（申万行业指数日线）| 2000 | 行业情绪的核心数据 |
| `top_list`（龙虎榜）| 2000 | 游资席位数据 |
| `stk_holdernumber`（股东人数）| 5000 | 筹码集中度 |
| `hm_detail`（沪深股通持仓明细）| 独立权限 | 北向个股持仓占比 |

**结论：** `moneyflow`、`hsgt_top10`、`margin` 三个接口需要 2000 积分（年捐助约 200 元），是动态驱动矩阵能跑起来的最低门槛。引擎应当在采集时检测权限，权限不足时在报告对应位置标注 `[数据源不可用，该因子跳过]`，不影响其他因子输出。

### 6.2 情绪指标数据来源补全

原蓝图在"情绪指标"部分只提到换手率，根据中金研究的左侧择时框架，A 股情绪体系应当更完整：

| 情绪指标 | 数据来源 | Tushare 接口 | 说明 |
|---------|---------|------------|------|
| 换手率 | Tushare | `daily_basic` | 近 5 日 vs 60 日均值 |
| 股权风险溢价 ERP | Tushare + FRED | `index_dailybasic` + FRED 国债收益率 | EP - 10Y 国债，求 5 年分位数 |
| 50ETF 认沽认购比 | Tushare | `opt_daily` | 期权市场情绪，左侧反转信号 |
| 融资余额变化 | Tushare | `margin` | 近 10 日融资余额增速 vs 流通市值增速 |
| 融券余额增速 | Tushare | `margin` | 负向情绪信号（与融资余额方向相反）|
| 创新高个股占比 | 引擎计算 | 基于 `daily` 全市场数据 | 大量创新高 = 过热，大量创新低 = 恐慌 |
| 北向近 5 日净流入 | Tushare | `moneyflow_hsgt` | 大幅净流入 = 偏热，大幅净流出 = 恐慌信号 |
| ETF 折溢价率 | 引擎计算 | `etf_nav` + `daily` | 连续极端折价 = 悲观过度；极端溢价 = 亢奋 |

**使用原则：** 这些情绪指标都是**左侧反转型**（多数有均值回复性），即极端高/低时才有参考价值，不是日常信号。引擎输出时应当给出分位数，而不是绝对值。唯一的趋势型指标是"前后成交额差异"（资金从下跌股转向上涨股），它是右侧跟随信号。

## 七、LAW 规范（LAW 10–16 完整版）

### LAW 10 — 分析提示必须具体，不得输出通用知识

每一节的「分析提示」必须满足：

1. **「为什么重要」** 说明该维度在定价逻辑中的具体传导路径（不是泛泛"很重要"）
2. **「常见分析误区」** 必须引用本次报告的至少一个具体数据点构造场景（不得写通用知识）

   ✅ 合规：「宁德时代 ROE 从 28%→24%，容易被误判为盈利能力下滑，
   但本次杜邦拆解显示净利润率稳定，下降来源于杠杆率收缩（健康信号）。
   误判根源：把"ROE 下降"直接等号"基本面恶化"，未做分解。」

   ❌ 违规：「ROE 下降时要注意可持续性，不能只看单年数字。」

3. **「下一步交叉验证」** 列出 1-3 个具体验证动作：查什么、和什么对比、期望看到什么结果

### LAW 11 — 研究问题卡必须来自结构化触发条件，不得凭空总结

**四类触发源（自动检测，优先排序）：**

**触发源 A — 变化驱动**
- 条件：近 20 日股价涨跌幅超 ±10% / 核心指标环比变化超 ±5pp / 近 30 日重大公告
- 问题格式：变化是什么 → 可能的驱动力 → 哪个解释尚不确定

**触发源 B — 估值位置驱动**
- 条件：PE/PB 处于历史 80% 分位以上或 20% 分位以下
- 问题格式：估值在哪 → 隐含的市场预期是什么 → 现实是否支撑这个预期

**触发源 C — 行业结构驱动**
- 条件：申万行业指数跑赢/跑输大盘超 5% / 近 30 日重大行业政策
- 问题格式：行业正在发生什么 → 对该公司的影响路径 → 是否改变投资逻辑

**触发源 D — 趋势结构驱动（新增）**
- 条件：股价处于 52 周高点 80% 以上或低点 20% 以下 /
  或 MA60 附近盘整超 15 个交易日
- 问题格式：当前趋势结构在哪 → 支撑这个位置的数据依据 → 什么条件会改变结构

**问题树固定格式：**

```
核心问题（1 句话，问号结尾，必须包含不确定性）
└── 子问题 ①（验证核心问题的关键数据点）
└── 子问题 ②（挑战核心问题的潜在反证）
└── 子问题 ③（若核心问题成立，对估值/股价的传导路径）

为什么这是好问题：[与估值定价的关系]
```

**禁止：** 宽泛问题 / 无不确定性的问题 / 含买卖判断的问题

---

### LAW 12 — 证据-结论映射强度标准

**三级统一定义：**

| 强度 | 符号 | 定义 |
|------|------|------|
| 强 | ✅ | 直接数据支持，来源可追溯，存在清晰因果链，排除了主要竞争性解释 |
| 中 | ⚠️ | 数据方向支持，但存在其他合理解释；或来源单一；或时效性有限 |
| 弱 | ❓ | 仅有时间相关性；或来源不可验证；或数据需进一步确认 |

**固定映射块格式：**

```
**结论：[一句话]**

支持证据：
  ✅ [数据 + 来源 + 时间]
  ⚠️ [数据 + 为何不够强]
  ❓ [数据 + 不确定点]

综合证据强度：[强 / 中 / 弱]
  [1-2 句解释：哪条证据最关键，什么使强度受限]
```

**禁止：** 输出结论不附映射块 / 模糊词不降强度 / 推断打 ✅

---

### LAW 13 — 候选解释不得伪装成归因结论，归因分析必须声明主导因子

**必须做的：**
1. 使用「候选解释」而非「原因」「因为」等确定性词汇
2. 每个候选解释附证据强度（LAW 12 三级标准）
3. 存在"无法解释的部分"时必须明确写出
4. 候选解释上限 5 条，按证据强度从强到弱排列
5. **新增：声明主导因子**

**主导因子声明格式：**

```
→ 当前主导因子：[对近期股价变化解释力最强的因子]
   依据：[1-2 句，说明为何是主导因子而非其他]
   主导因子的可持续性：短期（1-5 交易日）/ 中期（1-3 个月）/ 待观察
```

**禁止：** "因为 X 所以股价必然 Y" / 省略"无法解释的部分" / 时间相关性当因果

### LAW 14 — 分析框架核心题必须逐项回答，不得跳过

**每次报告必须回答 12 道核心必答题，数据不足时写明「数据不足：[缺少什么]」**

格式要求：
- 每个问题以该问题作为小标题
- 不得用通用叙述段落隐式覆盖多个问题（无法审查）
- 扩展题由触发条件动态激活（不强制）

### LAW 15 — 隐性预期差必须计算，无外部一致预期时仍须输出

**标准计算（简化戈登模型反推）：**

\[ g_{implied} \approx r - \frac{1}{PE} \]

其中 \( r = \) 无风险利率 + ERP，A 股默认取 10 年期国债收益率 + 6%（须标注来源/日期）。

**输出格式（固定）：**

```
当前 PE（TTM）：[X] 倍
10Y 国债收益率：[Y%]（来源/时间）
ERP 假设：6%（保守基准）
市场隐含增长率：约 [Z%]

对比：
  实际近 3 年营收/净利润 CAGR：[A%]
  一致预期（若有 WebSearch 数据）：[B%] / 或「无可靠数据，跳过」

解读：
  g_implied > 实际 CAGR → 市场定价偏乐观，需验证增长加速的依据
  g_implied < 实际 CAGR → 市场定价偏悲观，可能存在低估（需结合风险评估）
  g_implied ≈ 实际 CAGR → 定价基本反映历史增长，关注增长率拐点

⚠️ 本计算基于简化模型，对 PE > 50 的高成长/周期类公司参考价值有限，须额外说明。
```

### LAW 16 — 左侧/右侧判断必须呈现概率结构，不得输出单一结论

**左侧/右侧判断永远不输出"当前是左侧"或"当前是右侧"的确定性结论。**

原因：任何单边结论都隐含了对未来走势的预测，违反本 Skill 的基本约束。

**必须以以下格式输出：**

```
左侧概率的主要支撑依据：[列举，标注强度]
右侧概率的主要支撑依据：[列举，标注强度]
走势转变的触发条件：[列举，标注时间维度]
下一个重要观察节点：[时间 + 事件 + 关注指标]
```

**左侧参考指标体系（均值回复型，需给出分位数）：**
- 估值分位（PE/PB 历史 5 年分位）
- 股权风险溢价 ERP = EP - 10Y 国债，5 年分位数
- 50ETF 认沽认购比（高 = 悲观，左侧机会）
- 融券余额增速（高 = 看空情绪重，左侧参考）
- 北向近 5 日净流入方向（大幅流出 = 恐慌 = 左侧区域参考）
- RSI(14) 持续低于 30

**右侧参考指标体系（趋势跟随型）：**
- MA 多头排列（MA5 > MA20 > MA60 > MA120）
- 放量突破前期高点（量比 > 2）
- MACD 金叉 + 柱体连续放大
- 前后成交额差异：资金集中流入强势股（唯一的趋势型情绪指标）
- 行业板块突破前高且相对大盘强

## 十二、技术限制与风险

1. **动态驱动矩阵数据覆盖：** `moneyflow`、`hsgt_top10`、`margin` 需要 2000 积分。引擎应检测权限，权限不足时标注 `[数据源不可用，该因子跳过]`，不影响其他因子输出。

2. **ERP 计算依赖 FRED：** 项目已有 FRED 宏观数据采集，10 年期国债收益率（`DGS10`）可直接复用，无需额外接口。

3. **LAW 16 的执行防线：** 左侧/右侧输出最容易在 prompt 层被破坏，需要在 `SKILL.md` 和 `render.py` 两层同时约束，不能只靠 AI 自律。

4. **LAW 15 对高成长公司的局限：** g ≈ r - 1/PE 假设永续稳态增长，PE > 50 的公司结果失真，必须输出免责说明。

5. **50ETF 认沽认购比接口：** Tushare `opt_daily` 可获取，但需要 2000 积分。P0 阶段可暂缺，P2 阶段补入。
