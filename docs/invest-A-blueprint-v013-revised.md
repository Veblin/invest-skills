## 一、审计结论：蓝图在哪里还不够

原蓝图方向正确，但有三个系统性偏差，需要在重构前先纠正。

### 偏差 1：定位模糊，方法论包袱过重

原蓝图用了大量篇幅定义"学习提示卡""学习闭环""教的能力"，但你真正想要的是：

> **"帮我看清楚这家公司当前为什么在涨或跌，有没有我没看到的地方，左侧还是右侧，什么条件会让走势转变。"**

这是**投研助手**的定位，不是学习工具的定位。两者在方法论上高度重叠，但在报告语气、输出重心、结论导向上完全不同：

| 维度 | 学习工具定位 | 投研助手定位（你想要的） |
|---|---|---|
| 核心问题 | "这个指标代表什么" | "当前市场为什么这么定价" |
| 输出重心 | 逐节解释+常见误判 | 交叉验证+归因+分歧点+条件 |
| 结论态度 | "下一步验证什么" | "左侧/右侧概率分布，触发条件是什么" |
| 读者模型 | 需要被教的用户 | 能自己判断但需要多维信息交叉的用户 |

**修改方向：** 删除所有"学习提示卡""教的能力""初学者误判"的产品语言，把这些内容的实质保留下来，但包装成**研究备忘录**（Research Memo）的语气，而不是教材的语气。

---

### 偏差 2：缺少"解释当下"的核心能力

原蓝图的分析框架（行业/商业/财务/估值41题）本质上是**静态基本面扫描**，擅长回答"这家公司好不好"，但弱于回答：

- **为什么现在涨跌**（短中期驱动因子）
- **行业情绪和资金在哪边**（市场结构）
- **左侧还是右侧**（当前所处的趋势位置）
- **转变条件是什么**（催化剂识别）

这四个问题需要一个独立的、与静态基本面框架并列的**动态驱动分析层**。原蓝图把"股价变化候选解释"（LAW 13）和"研究问题卡"（LAW 11）都放了，但没有形成系统性的**多因子交叉验证框架**。

**修改方向：** 新增第五个分析模块 `E. 动态驱动分析`，作为与行业/商业/财务/估值并列的核心模块。这个模块专门回答"现在为什么、谁在驱动、左侧还是右侧"。

---

### 偏差 3：41题框架执行成本太高，会被跳过

LAW 14要求逐项回答41题，初衷好，但工程现实是：AI很可能用"数据不足，无法回答"填充大量题目，或者把答案写得极短以求"合规"。这会让分析质量下降。

**修改方向：** 把41题改为**分层激活**机制——核心必答题（每模块最重要的3题，共12题）+ 深度扩展题（根据触发条件动态激活）。这样既有框架约束，又有实际分析质量。

## 二、修订后的产品定位

```
invest-A 是一个多因子交叉验证的结构化投研助手。

它的核心能力是：
  ① 采集多渠道数据，交叉验证同一问题的不同数据源
  ② 归因分析：这家公司当前为什么涨或跌，驱动因子是什么
  ③ 多维度市场状态扫描：行业景气、资金态度、情绪、技术结构
  ④ 左侧/右侧概率判断：当前处于趋势的哪个阶段
  ⑤ 条件识别：什么事件/数据会让当前走势转变

它不做的：
  ① 不输出买卖建议，不输出目标价，不输出仓位建议
  ② 不把候选解释包装成确定性归因
  ③ 不把时间相关性声称为因果关系
```

### 多数据渠道交叉验证设计

这是原蓝图"多数据渠道交叉验证"定位的落地机制，贯穿全报告。

**交叉验证的三种结果，必须明确标注：**

```
🟢 印证（Convergence）：多个独立数据源指向同一结论
   → 证据强度提升，结论可信度高

🟡 分歧（Divergence）：不同数据源给出矛盾信号
   → 必须说明：哪两个数据源矛盾 / 可能的解释 / 哪个更可靠

🔴 缺口（Gap）：某个关键维度无法获得独立验证
   → 必须标注：缺少什么数据 / 该维度结论的可信度降低
```

**在报告中的具体应用位置：**

| 模块 | 交叉验证的数据对 |
|------|----------------|
| 财务质量 | 利润表净利润 vs 现金流量表经营现金流（利润质量验证） |
| 财务质量 | 营收增长 vs 应收账款增长（收入确认质量验证） |
| 估值 | PE 历史分位 vs PB 历史分位（估值位置一致性） |
| 资金态度 | 北向净流入 vs 主力大单净额（资金方向一致性） |
| 行业情绪 | 板块指数走势 vs 个股相对强弱（公司特异性验证） |
| 技术 vs 基本面 | MA 趋势方向 vs 近期业绩方向（趋势与基本面是否一致） |
| 估值 vs 资金 | PE 历史低位 vs 资金持续流出（低估陷阱识别） |

## 六、LAW 修订：三处重要更新

### LAW 10 修订：删除"学习"语境，保留实质

原条文中"初学者最常误判"的定义方向正确，但用词让报告看起来像教材。修订为：

**修订后 LAW 10 — 分析提示必须具体，不得输出通用知识**

每一节的「分析提示」（非"学习提示"）必须满足：

1. **「为什么重要」** 说明该维度在定价逻辑中的具体传导路径
2. **「常见分析误区」** 必须引用本次报告的至少一个具体数据点构造具体场景（不得写通用知识）
3. **「下一步交叉验证」** 列出 1-3 个可做的验证动作，具体到"查什么、和什么对比、期望看到什么"

---

### LAW 11 修订：研究问题卡增加"左侧/右侧"触发源

原三类触发源（变化/估值位置/行业结构）补充第四类：

**触发源 D — 趋势结构驱动**
- → 股价处于 52 周高点附近（80% 以上）或低点附近（20% 以下）
- → 或：日线/周线在关键均线上下反复（MA60 附近盘整超 15 个交易日）
- → 问题格式：当前趋势结构在哪 → 支撑这个位置的数据依据 → 什么条件会让结构改变

---

### LAW 16（新增）— 左侧/右侧判断必须呈现概率结构，不得给单一结论

**左侧/右侧判断永远不输出"当前是左侧"或"当前是右侧"的确定性结论。**

必须以以下格式输出：

```
左侧概率的主要支撑依据：[列举，标注强度]
右侧概率的主要支撑依据：[列举，标注强度]
走势转变的触发条件：[列举，标注时间维度]
```

原因：任何单边结论都隐含了对未来走势的预测，违反本 Skill 的基本约束。
呈现"概率结构+条件"让用户自己判断，这才是投研助手的正确输出方式。

## 七、分层激活的基本面问题集（替代原41题强制执行）

每个模块设定**3-4 道核心必答题**（每次报告必须回答），以及**扩展题**（由研究问题卡的触发条件激活）。

### 4a. 行业位置（核心 3 题）

**必答：**
- A-核心①：行业整体景气度——近期板块指数走势 + 产量/PMI 数据支持
- A-核心②：公司的行业竞争位置（龙头/挑战者/追赶者）+ 近年是否变化
- A-核心③：毛利率与行业中位数对比——差异能用议价权或规模解释吗？

**扩展激活条件：**
- 若触发源 C（行业结构驱动），激活：A4 替代威胁全部 2 题
- 若最近有行业政策事件（WebSearch），激活：竞争格局变化分析

### 4b. 商业质量（核心 3 题）

**必答：**
- B-核心①：护城河来源 + 近 3 年是否强化/侵蚀
- B-核心②：过去 3 年主要增长驱动力 + 是否仍在持续
- B-核心③：现金流模式——是否存在"利润好看但现金流弱"的结构性问题

**扩展激活条件：**
- 若 C3-⑧ 现金流覆盖比 < 0.8，激活：收入确认质量深度扫描
- 若业绩预告出现，激活：驱动力转换分析

### 4c. 财务质量（核心 4 题）

**必答：**
- C-核心①：近 3 年营收 CAGR + 近一年加速/减速
- C-核心②：杜邦拆解 ROE——三项各自贡献和趋势
- C-核心③：经营现金流/净利润覆盖比 + 应收/存货与营收增速对比
- C-核心④：扣非净利润/净利润——是否存在非经常性损益扭曲

**扩展激活条件：**
- 若任何报表风险信号触发，激活：对应的深度问题（来自原 C1-C4 全集）
- 若 ROE 近年变化超过 ±5pp，激活：杜邦完整拆解（所有 7 题）

### 4d. 估值与预期（核心 3 题 + LAW 15 计算）

**必答：**
- D-核心①：PE/PB 历史分位（5年窗口）——定性为昂贵/合理/偏低
- D-核心②：PE 与行业中位数对比——溢价/折价幅度及解释
- D-核心③：LAW 15 隐性预期差计算（g ≈ r - 1/PE）

**扩展激活条件：**
- 若触发源 B（PE/PB 处于历史 80%/20% 分位），激活：D4 全部预期差分析（3题）

## 十二、技术限制与风险

1. **动态驱动矩阵的数据覆盖：** 主力大单净额依赖 Tushare `moneyflow`，北向依赖 `hsgt_top10`，融资余额依赖 `margin`。部分接口有权限门槛，需要在报告中标注"该数据源不可用，以 [替代数据] 近似"。

2. **左侧/右侧判断的稳定性：** 这个输出极其容易被滥用成"建议"。LAW 16 的格式约束（呈现概率结构，不给单一结论）是关键防线，需要在 prompt 层强化执行。

3. **分层激活的触发精度：** 触发源 A/B/C/D 需要引擎在采集后自动判断，需要在 `collector.py` 输出中附带 `triggers` 字段传给 `render.py`。

4. **隐性预期差（LAW 15）对高成长公司的局限：** g ≈ r - 1/PE 假设永续稳态增长，对 PE > 50 的公司计算结果失真。需要在输出中附带"本估算对高增长公司参考价值有限"的说明。

5. **报告长度：** 9 模块 + 分层激活 + 交叉验证标注，预计 800-1000 行。P3 阶段需要做导航。

## 十三、总结

```
修订前的核心问题：
  ✗ 定位偏向"学习器"，但你要的是"投研助手"
  ✗ 缺少解释"当下为什么涨跌"的动态驱动分析
  ✗ 缺少左侧/右侧概率判断和走势转变条件识别
  ✗ 41 题强制执行，实际会被填充"数据不足"
  ✗ 多因子交叉验证没有系统化框架

修订后的核心能力：
  ✅ 候选解释 + 多因子驱动矩阵 + 交叉验证（解释当下）
  ✅ 市场结构分析（资金/情绪/板块，多源交叉）
  ✅ 左侧/右侧概率结构 + 走势转变触发条件（LAW 16）
  ✅ 12 核心题分层激活（基本面，不跑空）
  ✅ 归因分析增加主导因子声明

最优先做的一件事：
  实现 _section_dynamic_drivers() 和 _section_left_right_probability()，
  以及 collect_market_structure() 的数据采集。
  其他所有模块都是增量改进，只有这两个模块是从无到有的能力跃升。
```
